Залив на хостинг (без GitHub)
================================

1. ЧТО ЗАЛИТЬ
   Загрузите на хостинг всю папку проекта:
   - server.py
   - index.html
   - requirements.txt
   - Procfile
   - gunicorn.conf.py
   - runtime.txt
   - папка static/ (css, js, banners, images)
   - папка prompts/ (можно пустую)

   Не заливать: .env, .git, __pycache__, .DS_Store

   Python 3.14: проект настроен под Python 3.14 (runtime.txt, версии в
   requirements.txt). Менять не нужно — достаточно залить файлы как есть.

2. КЛЮЧИ API (OpenAI, Replicate)

   Вариант А — В панели хостинга есть "Переменные окружения":
   OPENAI_API_KEY     = ваш ключ OpenAI
   REPLICATE_API_TOKEN = ваш токен Replicate

   Вариант Б — Timeweb и хостинги БЕЗ переменных в панели:
   Создайте на сервере в корне проекта (рядом с server.py) файл .env
   с содержимым (без кавычек, без пробелов вокруг =):

   OPENAI_API_KEY=sk-ваш_ключ_openai
   REPLICATE_API_TOKEN=r8_ваш_токен_replicate

   Проект уже читает ключи из .env (python-dotenv). Файл .env не
   заливайте через публичный доступ — создайте его на сервере по FTP/SSH.
   PORT хостинг задаёт сам, менять не нужно.

3. ЗАПУСК
   Хостинг по Procfile выполнит: gunicorn server:app
   Либо вручную: pip install -r requirements.txt && gunicorn server:app

4. ДОМЕН
   В настройках приложения на хостинге укажите свой домен.
   SSL (https) обычно включается в панели хостинга.
   По умолчанию приложение отдаёт сайт в корне: заход на https://ваш-домен.ru/
   сразу открывает главную страницу и генерацию — подпапка не нужна.

4.1. ЗАГРУЗИЛ И ПОДКЛЮЧИЛ ДОМЕН — НЕ РАБОТАЕТ (ЧЕК-ЛИСТ)

   Что именно происходит?
   • Не открывается вообще (таймаут, «сайт недоступен»)
     → Домен должен указывать на сервер хостинга (A-запись или CNAME в DNS).
     → В панели хостинга домен должен быть привязан к этому проекту/серверу.
     → Подождите 5–30 минут после смены DNS.
   • Открывается пустая страница или «403 / 404»
     → Проверьте, что загружены index.html, server.py и папка static/.
     → Если хостинг только для статики — см. ниже.
   • Страница есть, но кнопки «Generate» дают 404
     → Бэкенд (Python) не запущен. См. пункт 5 и раздел 6 (Timeweb).

   Главное: какой у вас тип хостинга?
   • «Виртуальный хостинг» / «Сайт» / только HTML — этого недостаточно.
     Такой хостинг отдаёт только файлы. server.py там не выполняется,
     генерация не заработает. Нужен VPS или раздел «Приложения» (Python).
   • VPS или «Приложение (Python)» — тогда должен быть запущен gunicorn.
     В панели проверьте статус приложения; по SSH — процесс gunicorn и порт.

   На VPS после загрузки файлов нужно вручную:
   — установить зависимости (pip install -r requirements.txt);
   — создать .env с ключами;
   — запустить gunicorn (или настроить автозапуск через systemd/supervisor).

5. ЕСЛИ ПРИ НАЖАТИИ «GENERATE» ПОЯВЛЯЕТСЯ ОШИБКА 404
   — Убедитесь, что на хостинге запущен именно Python-бэкенд (gunicorn),
     а не только раздача статических файлов. Без server.py запросы
     /generate-ideas, /generate-prompts, /generate-images не обрабатываются.
   — Хостинг должен поддерживать запуск приложений (Python/Flask), а не только
     «веб-сайт» из HTML. Подойдут: Timeweb (приложение), Heroku, Railway,
     PythonAnywhere, VPS с gunicorn и т.п.
   — Если сайт открывается из подпапки (например site.com/myapp/), базовый
     путь подставляется автоматически. При необходимости задайте его вручную:
     в <html> добавьте атрибут data-api-base="/myapp" (замените /myapp на ваш путь).

6. TIMEWEB — КОНКРЕТНЫЕ ШАГИ

   На обычном «Виртуальном хостинге» Timeweb работает только статика (HTML/CSS/JS).
   Генерация будет давать 404. Нужен один из вариантов:

   Вариант А — Облачный сервер (VPS) Timeweb
   — Создайте облачный сервер (Ubuntu/Debian). Подключитесь по SSH.
   — Установите: Python 3.10+, pip, (опционально) nginx.
   — Залейте проект в папку, например /var/www/creative-factory/ (FTP/SFTP или git).
   — На сервере:
     cd /var/www/creative-factory
     python3 -m venv venv
     source venv/bin/activate
     pip install -r requirements.txt
     # Создайте .env с OPENAI_API_KEY и REPLICATE_API_TOKEN
     gunicorn --bind 0.0.0.0:8000 server:app
   — В панели Timeweb откройте порт 8000 или настройте nginx как прокси на 127.0.0.1:8000.

   Вариант Б — Timeweb «Приложения» / контейнеры (если доступно в тарифе)
   — В панели создайте приложение (Python), укажите корень проекта.
   — Команда запуска: gunicorn server:app (или как в Procfile).
   — Добавьте переменные окружения OPENAI_API_KEY и REPLICATE_API_TOKEN в настройках приложения.

   Ключи на Timeweb (если нет «Переменных окружения» в панели):
   — По SFTP создайте в корне проекта файл .env (рядом с server.py).
   — Содержимое: OPENAI_API_KEY=sk-... и REPLICATE_API_TOKEN=r8_...
   —    Файл .env не загружайте через публичный репозиторий.

7. RENDER.COM

   — Важно: в Dashboard сервиса зайдите в Settings → Start Command и укажите
     команду вручную. Не копируйте строку из Procfile (с «web:») — Render выполнит
     её целиком и выдаст «web: command not found». Введите ровно:

       python -m gunicorn server:app -c gunicorn.conf.py

   — Порт задаётся в gunicorn.conf.py из переменной PORT (по умолчанию 10000).
   — В Dashboard → Environment добавьте OPENAI_API_KEY и REPLICATE_API_TOKEN.
   — Если деплой падает с «Exited with status 1»: откройте логи деплоя (Events → Deploy → логи)
     и найдите строку с ошибкой Python (ModuleNotFoundError, ImportError и т.п.).
     Руководство: https://render.com/docs/troubleshooting-deploys

Готово к заливке.
